> "GPU 서버 → 10분 안에 AI 서버"라는 가치 제안은 매우 명확합니다. 실제로 CUDA/driver mismatch, container GPU passthrough 설정 등은 현장에서 시간 잡아먹는 고통 포인트이고, CK가 Proxmox + GPU 가상화로 수년간 직접 겪은 영역이라 기술적 신뢰도가 높습니다.

> Ubuntu/Red Hat이 성공한 건 Linux 커널 위에 패키지 관리 + 엔터프라이즈 지원이라는 명확한 레이어가 있었기 때문입니다. AI stack은 구성 요소(vLLM, Ollama, ChromaDB, Dify 등)가 아직 빠르게 바뀌고 있어서, distribution으로 묶는 순간 버전 호환성 관리가 엄청난 부담이 됩니다. 6개월마다 vLLM 메이저 버전이 바뀌고, 새로운 추론 엔진이 나오면 그때마다 distribution 전체를 재검증해야 합니다.

> CK가 만들려는 건 "AI Infrastructure Distribution"입니다. 베어메탈 GPU 서버를 받아서 10분 안에 AI 추론/RAG/운영이 가능한 서버로 만들어주는 올인원 플랫폼이죠. 리눅스 세계에서 Ubuntu나 Red Hat이 커널 위에 패키지 관리 + 엔터프라이즈 지원을 얹어서 "서버 OS 표준"이 된 것처럼, AI 서버 영역에서 같은 포지션을 잡겠다는 겁니다.

> 구성 요소는 크게 7개입니다. GPU 하드웨어 자동 감지/드라이버 설치, AI 추론 런타임(vLLM 등), 모델 관리(HuggingFace 연동), RAG 인프라(벡터 DB), OpenAI 호환 API 게이트웨이, GPU/latency/token 모니터링, 그리고 채팅 + 문서업로드 + 어드민 UI.

> 유지보수 부담이 가장 큰 리스크입니다. AI stack의 변화 속도가 너무 빠릅니다. vLLM, Ollama, ChromaDB, Dify 모두 월 단위로 breaking change가 나옵니다. Distribution이라는 건 이 모든 조합의 호환성을 보증하는 것이므로, 혼자서 이걸 따라가면 코드 작성보다 호환성 테스트에 시간을 더 쓰게 됩니다.
