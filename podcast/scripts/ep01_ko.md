# EP01 (KO) — Claude Code 핵심기능 5분 요약

안녕하세요. 오늘은 Claude Code를 “잘 쓰는 사람”들이 꼭 잡고 가는 네 가지 핵심 기능, **Subagents, Skills, Hooks, Plugins**를 5분 안에 정리해볼게요.
이 네 가지를 이해하면, “그때그때 프롬프트 잘 치기”에서 벗어나서 **내 작업 방식 자체를 시스템으로 만들 수** 있어요.

한 줄로 요약하면 이거예요:
- **Skills**는 “내가 자주 하는 일의 방법”을 저장해두는 거고,
- **Hooks**는 그 방법을 “자동으로 실행하거나 강제”하는 거고,
- **Subagents**는 일을 “컨텍스트를 분리해서” 시키는 거고,
- **Plugins**는 이걸 “묶어서 공유/배포”하는 방식이에요.

먼저 **Subagents**.
이건 쉽게 말해 “역할이 정해진 보조 Claude”를 하나 더 띄우는 느낌이에요.
예를 들어 탐색만 하는 에이전트를 따로 두면, 메인 대화는 깔끔하게 유지되고, 보조 에이전트가 파일 찾고 구조 파악하고 요약해주죠.
특히 Claude Code에는 **Explore** 같은 read-only 에이전트가 기본으로 있고, “검색/분석”은 거기 맡기면 컨텍스트가 덜 오염돼요.

다음은 **Skills**.
이건 “내 팀의 규칙”이나 “내가 자주 쓰는 작업 절차”를 **SKILL.md로 만들어서 재사용**하는 거예요.
예를 들어, PR 리뷰할 때 꼭 보는 항목이 있다면 체크리스트로 스킬을 만들 수 있고,
API 만들 때 에러 포맷, 네이밍 규칙 같은 걸 스킬에 넣어서 **매번 설명하지 않게** 만들 수 있어요.

스킬 만들 기준은 간단해요.
**반복되는데, 결과 품질이 중요하고, 사람이 자주 빼먹는다** — 이러면 스킬로 만들 가치가 커요.

그리고 배포 같은 “부작용 있는 작업”은, Claude가 마음대로 실행하면 위험하니까
`disable-model-invocation: true` 같은 옵션으로 **수동 호출 전용**으로 만드는 게 좋아요.

세 번째는 **Hooks**.
Hooks는 특정 타이밍에 자동으로 실행되는 “가드레일”이에요.
예를 들어 Bash에서 `rm -rf` 같은 위험한 명령이 나오면 **PreToolUse에서 차단**하게 만들 수 있고,
파일을 Write/Edit한 직후에 **자동 포맷/린트**를 돌리게 할 수도 있어요.
즉, 스킬이 “매뉴얼”이라면 훅은 “자동화/단속장치”에 가깝습니다.

마지막은 **Plugins**.
프로젝트 안의 `.claude/` 설정은 빠르게 실험하기 좋지만, 팀 전체에 공유하려면 번거롭잖아요.
플러그인은 Skills, Agents, Hooks를 묶어서 **버전 관리하고 배포**할 수 있는 형태예요.
대신 스킬 이름이 `/my-plugin:review`처럼 네임스페이스가 붙어서 좀 길어지는 건 감수해야 해요.
하지만 충돌을 막아주니까 팀 작업에서는 오히려 장점이 큽니다.

정리하면 이 순서로 시작하면 실전에서 잘 굴러가요:
1) `.claude/skills/`로 자주 하는 일을 스킬로 고정하고,
2) 위험하거나 자동화할 건 `.claude/settings.json`에 훅으로 가드레일을 붙이고,
3) 컨텍스트가 지저분해지면 서브에이전트로 분리하고,
4) 충분히 검증되면 플러그인으로 패키징해서 공유하기.

오늘 내용이 도움이 됐다면, 다음엔 “내가 실제로 자주 하는 작업 5개를 어떻게 스킬/훅/서브에이전트로 나눌지” 예시로 더 구체적으로 해볼게요.
감사합니다.
