# claude-code-best-practice

`claude-code-best-practice` is a large curated repository of Claude Code best practices and workflows. It organizes Claude Code into reusable primitives such as subagents, commands, skills, hooks, MCP servers, plugins, settings, checkpointing, routines, channels, and code review workflows.

## What the repo is good for
- Seeing how experienced users structure Claude Code workflows into repeatable patterns
- Tracking the ecosystem vocabulary around agentic engineering
- Borrowing practical ideas for agent orchestration, safety, and productivity
- Comparing Claude-native workflows with broader multi-agent tooling like ACP/ACPX

## Why it matters
The repo functions as a meta-reference for Claude Code usage. It is not just documentation; it is a living map of the ecosystem’s operating patterns and a useful signal source for agent-related workflows.

## GitHub signal angle
Because CK has GitHub logged into Chrome/CDP, stars on repos like this can be treated as explicit interest signals and saved into the wiki alongside other platform signals.
