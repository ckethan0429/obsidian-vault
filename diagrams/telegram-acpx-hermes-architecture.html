추천 아키텍처: Telegram + ACPX + Hermes

메신저를 조종석으로, 프로토콜을 실행면으로

사람은 Telegram에서 명령/승인만 하고, 실제 작업은 구조화된 런타임이 처리합니다. tmux는 디버깅/비상구로 남깁니다.
Batch / Queue / Resume / Cancel 중심

Telegram Bot

  • 사람이 직접 만지는 유일한 인터페이스
  • 명령, 승인, 알림, 상태 확인 담당

ACPX / ACP

  • 세션, 큐, 취소, 재개를 구조적으로 관리
  • PTY 스크래핑보다 안정적이고 확장성 높음

Hermes Workers

  • 실제 작업자 풀: 리서치, 수정, 테스트, 리뷰
  • tmux는 fallback으로만 유지
